## Confidence score

The combined confidence score is a weighted blend of the two detection signals:

```
confidence_score = 0.3 × score_signal_A + 0.7 × score_signal_B
```

Signal A measures structural patterns (type-token ratio and punctuation density). Signal B measures semantic/statistical patterns (perplexity / language model confidence). Both scores use the same scale: `1.0` = most AI-like, `0.0` = most human-like.

Because I noticed my program wasn't being accurate with confidence score, due to signal B being better at detecting human and AI than A, I changed the formula to weigh more towards signal B score.

## Rate Limiting

The `/submit` endpoint is limited to **10 requests per minute** and **100 requests per day**.

These limits were selected to balance usability and abuse prevention. A legitimate writer may submit multiple revisions of the same document while editing, but is unlikely to exceed ten submissions within a minute or one hundred submissions in a single day. The limits therefore allow normal interactive use while reducing the risk of automated scripts flooding the service with large numbers of requests, helping protect server resources and maintain fair access for all users.

Flask-Limiter is configured with in-memory storage for local development (`storage_uri="memory://"`) in `provenance_guard/extensions.py`. The limit is applied directly to the submit route:

```python
@submit_bp.post("/submit")
@limiter.limit("10 per minute;100 per day")
def submit():
    ...
```

## Audit Log

The audit log is exposed as structured JSON via `GET /log`. Each entry is a JSON object — not console output — with a `record_type` of either `decision` (classification) or `appeal`.

Every **decision** entry captures:

| Field | Description |
|---|---|
| `timestamp` | ISO-8601 UTC time of classification |
| `content_id` | Unique ID for this submission |
| `attribution_result` | Transparency label (attribution result) |
| `confidence` | Combined confidence score |
| `score_signal_a` | Structural signal score |
| `score_signal_b` | Semantic / LLM signal score |
| `appeal_filed` | `false` until an appeal is submitted for this content |

Every **appeal** entry captures the appeal timestamp, `content_id`, `appeal_id`, `creator_reasoning`, updated status, and a snapshot of the `original_decision`.
